# 교수님께 드리는 포트폴리오 코드 설명

이 저장소는 변기국의 AI/Data 포트폴리오 웹사이트입니다. 웹사이트를 열면 이력, 역량, 프로젝트, PDF 저장 기능을 볼 수 있고, GitHub에서는 그 화면을 만드는 코드와 문서를 함께 확인할 수 있습니다.

## 한 문장 요약

이 프로젝트는 `React`라는 웹 화면 제작 도구로 만든 정적 웹사이트입니다. 서버가 복잡한 계산을 하는 구조가 아니라, 미리 준비한 자기소개와 프로젝트 설명 데이터를 화면에 보기 좋게 배치하고 GitHub Pages로 배포합니다.

## 폴더를 쉽게 설명하면

| 위치 | 쉬운 설명 | 실제 역할 |
| --- | --- | --- |
| `src/data/portfolio.ts` | 포트폴리오의 원고 | 이름, 연락처, 학력, 프로젝트 내용, 성과 지표가 들어 있습니다. |
| `src/components/` | 화면을 구성하는 블록 | 상단 소개, 프로젝트 카드, 기술 스택, PDF 버튼 같은 조각입니다. |
| `src/pages/` | 실제 페이지 | 홈 화면과 FallAI 상세 페이지를 조립합니다. |
| `src/styles.css` | 디자인 규칙 | 글자 크기, 여백, 색상, 모바일 반응형을 정합니다. |
| `public/` | 그대로 배포되는 파일 | 이력서 PDF처럼 주소로 바로 열어야 하는 파일을 둡니다. |
| `docs/` | 설명 문서 | 교수님이나 협업자가 코드를 이해할 수 있게 적은 문서입니다. |

## 웹사이트가 만들어지는 흐름

1. `src/main.tsx`가 웹사이트 시작점입니다.
2. `src/App.tsx`가 현재 주소를 보고 홈 화면인지 상세 화면인지 고릅니다.
3. `src/pages/Home.tsx`가 홈 화면을 조립합니다.
4. `src/data/portfolio.ts`에 있는 포트폴리오 내용을 각 컴포넌트에 전달합니다.
5. 컴포넌트가 받은 내용을 카드, 표, 버튼, 문단으로 화면에 보여줍니다.
6. `npm run build`를 실행하면 GitHub Pages에 올릴 수 있는 완성 파일이 `dist/`에 만들어집니다.

## 주요 화면

### 1. 첫 화면

`Hero.tsx`가 담당합니다. 이름, 지원 분야, 주요 버튼을 보여줍니다. 버튼에는 프로젝트 보기, PDF 저장, 이력서 PDF, 연락하기가 있습니다.

### 2. 포트폴리오 요약

`ProfileSnapshot.tsx`가 담당합니다. 교수님이나 채용 담당자가 길게 읽기 전에 핵심 수치와 직무 적합성을 먼저 볼 수 있도록 만들었습니다.

### 3. 프로젝트 카드

`FeaturedProjects.tsx`가 담당합니다. 대표 프로젝트인 FallAI를 짧게 요약하고, 어떤 성과를 증명하는 프로젝트인지 보여줍니다.

### 4. FallAI 상세 페이지

`FallAIPage.tsx`, `MainProject.tsx`, `ProjectDetail.tsx`, `FallAIScreenPreview.tsx`가 함께 담당합니다. AI 낙상 판단 보조 시스템의 문제 정의, 접근 방식, 화면 예시, 기술 스택을 설명합니다.

### 5. PDF 저장

`PortfolioPdf.tsx`와 `PdfExportButton.tsx`가 담당합니다. 웹사이트 내용을 PDF로 출력하기 좋게 별도 문서 형태로 다시 구성합니다.

## 이 코드에서 강조하고 싶은 점

- 내용을 코드 안에 흩뿌리지 않고 `portfolio.ts`에 모아 두어 수정이 쉽습니다.
- 화면을 작은 컴포넌트로 나누어 어떤 부분이 무엇을 담당하는지 찾기 쉽습니다.
- GitHub Pages에 올릴 수 있는 정적 사이트라 서버 비용 없이 공개할 수 있습니다.
- PDF 저장 기능을 넣어 웹사이트와 제출용 문서를 함께 관리할 수 있습니다.
- README와 docs 문서를 추가해 코드를 모르는 사람도 구조를 따라갈 수 있게 했습니다.
